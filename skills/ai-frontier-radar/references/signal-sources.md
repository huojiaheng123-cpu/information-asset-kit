# Signal Sources

Use these source clusters when building deeper AI frontier scans. Prefer primary sources first, then use aggregators as discovery aids.

## GitHub

- Trending: daily and weekly trending pages, language filters, topic pages such as `artificial-intelligence`, `ai-agents`, `llm`, `mcp`, `rag`, `text-to-speech`, `computer-use`, and `coding-agent`.
- Repository health: stars over time, forks, contributors, recent commits, release tags, open issues, discussions, docs quality, examples, CI, package publish history.
- Expansion signals: plugins, MCP servers, VS Code extensions, CLI wrappers, Docker support, hosted demos, integrations with Claude Code, Codex, Cursor, GitHub Copilot, JetBrains, or browser automation.

## Hugging Face

- Model search: trendingScore, downloads, likes, createdAt, lastModified.
- Spaces: working demos are stronger than model cards alone.
- Model cards: check license, benchmarks, inference examples, hardware requirements, and fine-tuning data disclosure.
- Watch for: small models beating older large baselines, multimodal demos, real-time voice/video, tool-use models, VLMs for GUI/computer use, and domain models.

## Papers And Labs

- arXiv and Papers with Code for new methods, but prefer items with code, demos, or replication.
- Lab blogs and docs: OpenAI, Anthropic, Google DeepMind, Meta AI, Microsoft, GitHub, Mistral, xAI, Alibaba/Qwen, DeepSeek, ByteDance, Tencent, Baidu, Moonshot, Zhipu.
- Evaluate whether a paper introduces a primitive, a benchmark, a dataset, or a practical workflow.

## Product And Market Chatter

- Official changelogs: OpenAI, Anthropic, GitHub, Cursor, Windsurf, Vercel, Cloudflare, Hugging Face, LangChain, LlamaIndex.
- Developer discussions: Hacker News, Reddit, GitHub issues/discussions, X threads when accessible.
- Strong signals: developers complaining about a repeated pain, teams posting migration stories, new job titles or workflows forming around a capability.

## Noise Filters

- Beware repos with huge stars but little code, no install path, no releases, no issue response, or copied demos.
- Beware benchmark-only claims without runnable examples.
- Beware "agent" projects that are prompt wrappers with no durable memory, tool safety, evals, or workflow integration.
- Beware star spikes from giveaways, bot activity, or one viral post unless usage signals follow.

## Opportunity Angles

- Tool wrapper: package a hard-to-use capability behind a simple CLI, extension, or MCP server.
- Vertical workflow: apply a general AI capability to a narrow role with clear ROI.
- Infrastructure: memory, eval, observability, sandboxing, cost control, data pipelines, permissions, and compliance.
- Distribution: templates, guides, demos, localized versions, benchmark comparisons, and integration recipes.
